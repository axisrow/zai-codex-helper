<objective>
Deliver the injectable frozen `Paths` dataclass (ROADMAP Phase 2, PKG-03) through which every filesystem path the tool touches is resolved from a single injected `home` — so no test and no production call ever hard-codes or corrupts the developer's real `~/.codex`, `~/.zshrc`, or `~/Library/LaunchAgents/`. This is the root configuration object of the project's "compiler whose target is the user's filesystem" architecture.

Purpose: SC-1 (from_home resolves all paths under one home) + SC-2 (a unit test round-trips them and provably never touches real $HOME). Phase 2 ships the object + its unit test ONLY — no wiring into main()/build_parser()/handlers (D-24; real wiring lands in Phase 4 when backends accept Paths).

Output: `src/zai_codex_helper/services/paths.py` (the Paths dataclass + from_home + default), a thin `services/__init__.py` re-export, and `tests/test_paths.py` (green unit tests). All stdlib (dataclasses + pathlib) — zero new runtime dependencies.
</objective>
