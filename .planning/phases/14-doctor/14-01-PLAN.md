<objective>
Deliver `zai-codex-helper doctor` — a READ-ONLY diagnostic that walks the entire Codex ⇄ Moon Bridge ⇄ Z.ai chain link-by-link and prints a colored verdict (`[✓]`/`[!]`/`[✗]`) plus a "To fix:" hint for every non-pass, exiting non-zero only on a hard `✗`. This is the observability/troubleshooting companion to `setup`/`install-service` and the LAST Phase 1 stub to become real.

Purpose: When Z.ai isn't working, `doctor` pinpoints the broken link (e.g. port open but auth wrong → port `✓`, `/v1/models` `✗`). It composes prior-phase detection (Phase 10 binary, Phase 13 lifecycle, Phase 9 backends, Phase 8 status) and adds the first runtime use of httpx (hard-timeout HTTP probes) and Codex-Desktop pgrep detection.

Output: `services/doctor.py` (run_doctor + 9 checks + CheckResult + ANSI color helpers + TTY gate), the real `_handle_doctor` in `cli/parser.py` (emptying the stub loop), and `tests/test_doctor.py` (pytest-httpserver + mocked runner).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| doctor → local Moon Bridge (127.0.0.1:38440) | httpx GET /v1/models + POST /v1/responses cross into the locally-running proxy; a hung/corrupt proxy could stall doctor or return misleading status |
| doctor → filesystem (~/.codex/*) | read-only stat/read of binary, yml, models_cache, config.toml; a corrupt file (malformed YAML, non-object JSON) must be reported, not crash doctor |
| doctor → subprocess (pgrep, launchctl via verify_service_loaded) | runner seam; pgrep inspects process table, launchctl inspects agent state — both via the injectable runner, never raising into the caller |
| doctor output → user terminal | ANSI color codes written to stdout; must degrade to plain markers when piped (not a TTY) so logs/captured output stay readable |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-14-01 | Information Disclosure | doctor HTTP probes (POST /v1/responses) | medium | mitigate | The POST probe sends a MINIMAL glm-5.2 payload to the LOCAL Moon Bridge only (127.0.0.1:38440), never to upstream Z.ai and never including ZAI_API_KEY in the doctor code (the key lives in moonbridge-zai.yml which Moon Bridge reads). doctor never logs the key. Hard timeout bounds exposure to a malicious local listener. |
| T-14-02 | Denial of Service | httpx probes vs. hung Moon Bridge | medium | mitigate | Single httpx.Client with HARD timeout (5.0s connect+read, D-90) on BOTH probes so a hung/stuck proxy cannot stall doctor; the probe fails fast and is reported as a ✗. |
| T-14-03 | Tampering | doctor writes (scope discipline D-94) | high | mitigate | doctor is READ-ONLY by contract: no writes, no launchctl bootstrap/bootout, no build, no models_cache write. Enforced by a unit test asserting the tmp HOME file set is byte-identical before/after a full run. |
| T-14-04 | Repudiation | doctor exit code / output | low | accept | doctor prints its own colored verdicts and owns its exit code (0 unless ✗); the per-check CheckResult trail is the audit surface — sufficient for a diagnostic CLI. No further logging required at v1. |
| T-14-05 | Spoofing | pgrep -x Codex (DIAG-03) | low | accept | A process literally named "Codex" is treated as Codex Desktop; the check is a WARN (not fail), so a false positive only surfaces a harmless staleness hint. No auth decision rides on it. |
