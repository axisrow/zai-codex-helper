<objective>
Deliver the `zai-codex-helper status` command — a strictly read-only observability
command that prints, at a glance: the current default provider (Z.ai vs OpenAI),
the resolved config file paths in play (exists/missing), and the installed
package version. It performs NO writes — ever — and exits 0 on a parseable config
(or a missing one) and non-zero on a broken (unparseable) one.

Purpose: This is the observability companion to the Phase 7 Core Value. After
`use zai`, a user runs `status` to confirm Z.ai is active without hand-reading
config.toml. It reuses Phase 5's `TomlBackend.read` (read-only), Phase 2's
`Paths` (path resolution), Phase 1's `__version__`, and Phase 6's provider keys —
no new IO primitives. It is the single deliverable for ROADMAP Phase 8 and
satisfies requirement PROV-05.

Output:
- A real `_handle_status(args) -> int` in `cli/parser.py` replacing the Phase 1
  `_stub("status")` wiring.
- A pure provider-detection helper in `services/status.py` (no IO; takes a parsed
  doc and returns a provider descriptor) — D-54 allows this; it makes detection
  unit-testable in isolation.
- `tests/test_status.py` pinning both ROADMAP Success Criteria.

Honors locked decisions D-50..D-55 verbatim (see each task action and the
must_haves above).
</objective>
