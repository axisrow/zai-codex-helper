## Phase Goal

**As a** developer using zai-codex-helper,
**I want** every file mutation to land crash-safely via temp + fsync + os.replace with a configurable permission mode,
**so that** an interrupted write never corrupts my Codex config and my API key lands at `0600`.

<objective>
Deliver the single crash-safe file-write helper that every later backend (Phase 4 `ConfigBackend` ABC, Phase 5 `TomlBackend`, Phase 9 `YamlBackend`/`JsonBackend`/`ShellBackend`/`PlistBackend`) delegates to for disk mutations. Writes go to a temp file **in the same directory** as the destination, are `os.fsync`-ed, then `os.replace`-d atomically. An interrupted write (crash, power loss, Ctrl-C, exception) never leaves a half-written config at the destination and never orphans a temp file. The helper accepts a `mode` parameter: `None` preserves the existing/umask mode (for `config.toml`), `0o600` produces a restricted-permission destination (for secrets like `moonbridge-zai.yml`).

This is the write-boundary counterpart to Phase 2's pure `Paths` object: `Paths` resolves *where* (D-22 ‚Äî pure, no IO), this helper decides *how* (crash-safe + permissions). It lives at the `backends/` IO boundary (D-09), not in pure `services/`.

Purpose: Pin CONF-01 ("Atomic write for all mutations: temp + fsync + os.replace; 0600 for secrets") as a single reusable primitive so no ad-hoc `open(...).write()` ever appears elsewhere. Phase 4's `ConfigBackend` imports this helper without rework.
Output: `src/zai_codex_helper/backends/_atomic.py` + `tests/test_atomic_write.py` (green, ruff-clean).
</objective>
