<objective>
Detect Go / brew / Moon Bridge binary presence and offer to install missing
toolchains only with explicit consent.

**As a** developer running the helper toward a Moon Bridge build,
**I want to** know whether Go, brew, and `~/.codex/moon-bridge` are present
(with the correct brew arch resolved) and be guided to install anything
missing,
**so that** the tool never silently auto-installs a system toolchain and never
breaks on the "wrong" brew path.

Purpose: This is the prerequisite for Phase 11 (build-from-source) and Phase 12
(setup orchestrator). Phase 11's `go build` needs to know Go is present;
Phase 11/12's `brew install go` suggestion needs to know which brew. Phase 10
only DETECTS and OFFERS — it writes nothing and builds nothing (D-68).

Output:
- `src/zai_codex_helper/services/deps.py` — DepResult dataclass + detect_go /
  detect_brew / detect_moonbridge_binary / offer_install.
- `src/zai_codex_helper/services/io.py` — shared confirm() helper.
- `tests/test_deps.py` — unit tests pinning SC-1/SC-2/SC-3.
</objective>
