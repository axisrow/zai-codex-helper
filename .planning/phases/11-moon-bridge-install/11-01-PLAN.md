<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| helper → filesystem (`~/.codex/moon-bridge`) | The orchestrator writes a built executable to the user's codex dir and chmods it 0755. A tampered clone path or a malicious repo substitution could plant an arbitrary binary. |
| helper → network (`git clone github.com/ZhiYi-R/moon-bridge`) | The clone crosses the trust boundary to the public internet. The pinned SHA (not main) is the integrity control — a fixed commit cannot drift under an attacker who later compromises the repo's default branch. |
| helper → subprocess (`git`, `go`) | The orchestrator shells out to system toolchains. argv is constructed from module constants (not user input), so command injection is not possible, but a malicious PATH could substitute a trojan `git`/`go`. |
